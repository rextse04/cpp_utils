\doxysubsection{Detailed Description}
This file provides utilities for Rust-\/like dynamic dispatch, as a partial replacement for virtual methods in standard C++. The major difference between this system and that in the language is that rather than embedding a pointer to vtable in each object, it is stored in each pointer to an object. The advantages of this approach are that it reduces the footprint of \char`\"{}dynamic objects\char`\"{} and one level of indirection on each dynamic dispatch.

Important concepts and remarks\+:
\begin{DoxyItemize}
\item An {\bfseries{Interface}} is any class type that defines what its {\bfseries{implementers}} must implement. Under the hood, the interface is directly embedded in its associated vtables, and so there are few limits on what the class type can be. However, practically, you would probably want an interface to be an aggregate for the provided macros to be usable.
