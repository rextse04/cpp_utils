Detailed Description

Key concepts and conventions:

  • A Result is a type that either has a member type alias type, or a constexpr static member value, named TypeResult and ValueResult respectively. For example, std::integer_constant instances are ValueResults. This can be seen as a generalization of types obtained by applying Traits.
  • An ErasedResult is not a Result. It has a member type alias result, which is defined to be a TypeResult or a ValueResult. It can be inferred to a Result, however.
  • A Trait is a template that accepts a fixed or variadic number of template type arguments. Any instance of a Trait should be a Result, or the program should be ill-formed (which is usually guaranteed by setting static_asserts). In addition, valid instances of Trait should either all be TypeResults, or all be ValueResults. They are named TypeTraits and ValueTraits respectively.
  • A MetaTrait is a template whose valid instances have an inner template which satisfies Trait.
